Qu es R ?

  • R es un dialecto de S

Qu es S ?

  • S es un lenguaje de programacin desarrollado por Jhon Chambers en Bell Labs en 1976.

  • Inicialmente se gener como un entorno estadstico interno (implementado en Fortran).
  • En 1988 es reescrito en lenguaje C (versin 3)
  • En 1998 se lanza la versin 4, que actualmente se maneja.
  • Bell Labs –> Insightful Corp (licencia, lenguaje, 2 M.)
  • En 2004 Insightful aade GUIs y varas caractersticas, S-PLUS

  • En 2008 Insightful Corp –> TIBCO (lenguaje, 25 M)

Historia de R

  • Fue desarrollado en 1991 por Ross Ihaka y Robert Gentleman en la Universidad de Auckland.
  • En 1993 se anuncia al pblico en general.
  • En 1995, Martin Mchler convence a Ross y Robert de distribuir a R mediante la licencia GNU (General Public License).
  • EN 1996, se crean las listas de correo pblicas R-help y R-devel.
  • En 1997 se forma el R Core Group o R Core Team, encargado de controlar el cdigo fuente de R.
  • En el 2000 se publica la primera versin de R 1.0.0.
  • En marzo 2015 se publica la versin 3.1.3.

Instalando paquetes

Una de las principales ventajas de R es su capacidad para incrementar su funcionalidad mediante la incorporacin de nuevos paquetes o libreras.

  • Para instalar un paquete se utiliza la linea de cdigo:
install.packages("pckgname")

Ciertos paquetes de R requieren la instalacin de otros paquetes adicionales debido que entre estos comparten algunas funciones, por lo cual es necesario adicionar el siguiente parmetro:

install.packages("pckgname", dependencies=TRUE)

En el caso que se desee instalar un lista de \(n\) paquetes planteamos la siguiente solucin:

paquetes <- c("pckg_1","pckg_2", ... , "pckg_n")
install.packages(paquetes, dependencies=TRUE)
# funciones apply
# sapply(X=paquetes, FUN=install.packages, dependencies=TRUE)

Ejemplos

  • Para instalar el paquete de nombre "devtools", utilizamos la lnea de cdigo:
install.packages("devtools", dependencies = TRUE)
  • Para instalar los paquetes "foreign", "forecast", "gWidgets", utilizamos:

Vectores

  • La estructura de datos bsica de R son los vectores, estos se dividen en:
    • Vectores atmicos y Listas
  • Presentan tres propiedades comunes:
    • Tipo (Qu es?). typeof()
    • Longitud (Nmero de elementos) length()
    • Atributos (Metadata). attributes()
  • Para probar que una estructura de datos es un vector (atmico o lista) se utiliza la funcin:
is.vector(x)
  • is.vector(x) retorna TRUE si x es un vector atmico o una lista.

Vectores atmicos

  • Los elementos de un vector atmico son del mismo tipo (tipos comunes):
    • logical
    • integer
    • double (comunmente llamados numeric)
    • character
  • Un vector es creado mediante la funcin c(), e ingresamos a la i-sima componente con x[i]
dbl_var <- c(1, 2, -1) 
int_var <- c(3L, 7L, 1L) # con el sufijo L se declaran nmeros enteros
log_var <- c(FALSE, TRUE, F, T) # TRUE y FALSE (T, F) se usan para crear vectores lgicos
chr_var <- c("Source", "Stat", "Lab")
# tercera componente de dbl_var
dbl_var[3]
c(1, c(2, c(3,4))) ; c(1, 2, 3, 4) # los vectores pueden ser anidados
## [1] 1 2 3 4
## [1] 1 2 3 4

Coercin

  • Los elementos de un vector son del mismo tipo, es por eso que si combinamos tipos diferentes, sern coercionados al tipo ms flexible dado por la jerarqua:

    • character <= double <= integer <= logical
x <- c("ssl", 1, TRUE)
str(x)
##  chr [1:3] "ssl" "1" "TRUE"

  • Se usa las funciones "as" para coercionar un vector a un determinado tipo: as.character(), as.double(), as.integer(), as.logical()

  • Las funciones +, abs, log, etc coercionan a double o integer

  • Las funciones &, |, any, all, etc coercionan a logical

x <- c(1L, 3L) ; y <- c(2L, 5L) ; str(x + y)
##  int [1:2] 3 8

Coercin 

x <- c(FALSE, TRUE, FALSE, TRUE) 
as.numeric(x) # TRUE coerciona a 1 y FALSE a 0
## [1] 0 1 0 1
sum(x) # nmero total de TRUEs 
## [1] 2
mean(x) # proporcin de TRUEs
## [1] 0.5

Listas

  • Una lista puede contener elementos de cualquier tipo.
  • Para construir una lista se utiliza la funcin list() en lugar de c().
x <- list(1:3, c("Source", "Stat", "Lab"), c(TRUE, FALSE), c(1.3, 4.5))
# El operador `:` se usa para generar secuencias enteras
str(x)
## List of 4
##  $ : int [1:3] 1 2 3
##  $ : chr [1:3] "Source" "Stat" "Lab"
##  $ : logi [1:2] TRUE FALSE
##  $ : num [1:2] 1.3 4.5
is.atomic(x)
## [1] FALSE
# coercionamos un objeto a una lista
as.list(c("ssl", 4, 0.5))

Listas 

x <- list(1:3, c("Source", "Stat", "Lab"), c(TRUE, FALSE), c(1.3, 4.5))
typeof(x)
## [1] "list"
# probamos si x es una lista
is.list(x)
## [1] TRUE
# pasamos a vector considerando las reglas de coercin
unlist(x)
##  [1] "1"      "2"      "3"      "Source" "Stat"   "Lab"    "TRUE"  
##  [8] "FALSE"  "1.3"    "4.5"

Listas

  • Una lista es un vector recursivo, es decir una lista puede contener otra lista (diferencia principal con los vectores atmicos)
x <- list(list(list("Source", "Stat")))
str(x)
## List of 1
##  $ :List of 1
##   ..$ :List of 2
##   .. ..$ : chr "Source"
##   .. ..$ : chr "Stat"
# son equivalentes. un vector no puede contener otro vector
c(1, c(2, c(3,4)))  ;  c(1, 2, 3, 4)
## [1] 1 2 3 4
## [1] 1 2 3 4

Elementos de una lista

  • Para acceder al elemento i de la lista se utiliza x[i], en cambio para acceder al objeto que contiene el elemento i se utiliza x[[i]] o x$nomb_obj.
x <- list(1:3, c("Source", "Stat", "Lab"), nomb_obj=c(TRUE, FALSE), c(1.3, 4.5))
x[4]
## [[1]]
## [1] 1.3 4.5
x[[4]]
## [1] 1.3 4.5
x$nomb_obj
## [1]  TRUE FALSE

Missing Values

  • Los valores prdidos en R se denotan por NA, NaN.
  • NA Not Available. Dato prdido.
  • NaN Not a Number. Resultado de una indeterminacin.
    • NaN es NA, el reciproco no se cumple.
x <- c(Inf*0, 3.2, NA, 5, NA)
x
## [1] NaN 3.2  NA 5.0  NA
x <- c(-1, 0/0, NA, 5, -6)
x
## [1]  -1 NaN  NA   5  -6

Missing Values

  • is.na(). prueba si un objeto es NA.
  • is.nan(). prueba si un objeto es NaN.
x <- c(Inf*0, 3.2, NA, 5, NA)
is.na(x)
## [1]  TRUE FALSE  TRUE FALSE  TRUE
x <- c(-1, 0/0, NA, 5, -6)
is.nan(x)
## [1] FALSE  TRUE FALSE FALSE FALSE
